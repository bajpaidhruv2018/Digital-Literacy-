# Case Study: Ransomware Attack on a Student Developer

[cite_start]**The Crime:** Ransomware is a type of malicious software designed to block access to a computer system or files by encrypting them until a sum of money is paid[cite: 78].

**The Scenario:** Rahul, a second-year engineering student, needed an expensive proprietary software tool for a final project. Instead of using the university's lab computers, he downloaded a "cracked" version from an unverified peer-to-peer forum. [cite_start]The executable file contained a hidden ransomware payload[cite: 78, 79].

**Step-by-Step:** When Rahul ran the installer, it bypassed his basic Windows Defender setup. It silently executed in the background, using AES-256 encryption to lock his entire `C:\Users\` directory. This included all his local code repositories, uncommitted hackathon projects, and final assignment documents. [cite_start]Upon reboot, his wallpaper was replaced with a ransom note demanding $500 in Bitcoin within 48 hours to receive the decryption key[cite: 78].

[cite_start]**The Target:** College students are highly susceptible to ransomware because they frequently download pirated software, game mods, or cracked tools to save money, often while lacking robust, segmented cloud backup systems[cite: 78].

**The Consequences:** Unable to afford the ransom—and knowing that paying rarely guarantees data recovery—Rahul had to wipe his drive completely. [cite_start]He permanently lost hundreds of hours of coding work, resulting in a failed semester project and the total loss of his local digital portfolio[cite: 78].
